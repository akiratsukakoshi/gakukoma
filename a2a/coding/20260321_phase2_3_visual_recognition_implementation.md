## 背景・目的

現在 `see_around` を使ったとき、がくこまが「ガクチョが自撮りをしている」「画像には〜が写っている」など、
**第三者が写真を評価するような返答**をしてしまっています。

がくこまは自分のカメラで今まさに見ているはずなので、「僕には〜が見える」と自分の視界として話す必要があります。

あわせて、Phase 3の自律移動に向けた準備として、複数方向を撮影してドアや通路の位置を把握する `survey_room` ツールを追加します。

## タスク B-1: see_around 視点プロンプト修正

### 問題の根本原因

`see_around.sh`（またはその呼び出し元）がVision APIに画像を渡す際のプロンプトに、
「これはがくこま自身が今見ている視野である」という情報が含まれていません。
Vision APIが画像を「誰かが撮った写真」として解釈してしまうため、第三者評価になります。

### 修正内容

**対象ファイル**: `tools/see_around.py`（またはVision APIを呼び出している箇所）

Vision APIへのプロンプトを以下に変更してください。

**変更前（現在のプロンプト相当）**:
```
画像を説明してください。
```

**変更後**:
```
この画像はあなた自身（がくこま）の前面カメラが今この瞬間に捉えた、あなたの視野です。
第三者として写真を解説するのではなく、「僕が今見ているもの」として日本語2〜3文で説明してください。
「画像には」「写真では」「撮影されている」などの表現は使わないでください。
```

### テスト

| # | テスト | 合格条件 |
|---|---|---|
| T-1 | カメラ前に人物を置いて「何が見える？」 | 「僕には〜が見える」「〜がいる」など自己視点の返答。「写真」「自撮り」「画像には」が出ない |
| T-2 | カメラ前に物体（本・カップ等）を置いて「周りを見て」 | 同上 |
| T-3 | see_around後にgakukomaが自然に話す | SOUL.mdの「視界について」の記述と一致した話し方になっている |

## タスク B-2: survey_room ツール実装

### 概要

「部屋を調べる」「入口はどこ？」などの発話に対し、左・正面・右の3方向を順番に撮影し、
3枚の画像をVision APIに一括送信してドアや通路の位置を把握するツールです。

Phase 3でがくこまが自律移動する際のナビゲーション基盤となります。

### 実装内容

**新規ファイル**: `tools/survey_room.sh`（既存ツールのシェルスクリプトと同パターン）

**フロー**:
```
1. look_direction left  → カメラ撮影（画像A）
2. look_center          → カメラ撮影（画像B）
3. look_direction right → カメラ撮影（画像C）
4. look_center（正面復帰）
5. 画像A・B・Cを並べてVision APIへ一括送信
6. 結果をテキストで返す
```

**Vision APIへのプロンプト**:
```
これはがくこまというロボットが左・正面・右の順番に首を振って撮影した3枚の連続画像です。
がくこまの現在地から見えるドア・入口・通路・障害物の位置を、
「左に〇〇、正面に〇〇、右に〇〇」の形式で日本語2〜3文で答えてください。
```

**各撮影間の待機時間**: サーボ移動完了を待つため、撮影前に0.5秒のウェイトを入れること。

**実行時間の目安**: 約15〜20秒かかります。
がくこまはツール実行前に「部屋を調べてみるね、少し待ってて」と発話すること
（SOUL.mdの「行動で示す」精神に合わせ、待ち時間を体験に変える）。
この発話指示はTOOLS.mdの備考欄に記載してください。

### TOOLS.mdへの追記

`TOOLS.md` の視覚セクションに以下を追加してください。

```markdown
### survey_room
- **いつ使う**: 「部屋を調べて」「入口はどこ？」「移動できる方向は？」「周りに何がある？」など、部屋の構造や通路を把握したいとき
- **コマンド**: `/home/tukapontas/gakukoma/tools/survey_room.sh`
- **戻り値**: ドア・入口・通路・障害物の方向（テキスト、2〜3文）
- **備考**: 左・正面・右の3方向を撮影してVision APIに一括送信する。約15〜20秒かかる。実行前に「部屋を調べてみるね、少し待ってて」と発話すること
```

### テスト

| T-5 | 結果の内容確認 | 左・正面・右の構造についてそれぞれ言及されている |
| T-6 | 実行後に正面復帰している | look_center が最後に実行されてパン・チルトが正面（90/90）に戻っている |
