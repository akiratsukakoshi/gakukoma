## 2. 実装の概要

### 2.1 新規モジュール: `led_controller.py`
- `gpiozero.RGBLED` をラップしたクラスを作成。
- **仕様通りの色・点滅制御**: `RGBLED.blink()` の `background=True` 機能を利用し、CPUに負荷をかけずに非同期で点滅を継続。
- **排他制御の簡略化**: 新しい状態を設定する直前に `self.led.off()` を呼ぶことで、実行中の点滅スレッドを安全に停止させ、色の混ざりを防止。

### 2.2 `voice_loop.py` への統合
- **ステート同期**: `self.state` が更新される全ての箇所（8箇所）に `self.led.set_state("xxx")` を追加。
- **リソース管理**: `KeyboardInterrupt` をキャッチした際に、必ず `self.led.close()` を呼び出し、GPIOピンの状態をリセット（消灯）して終了するよう修正。
- **堅牢性**: 予期せぬエラーで `self.led` が初期化されなかった場合を想定し、解放時に `hasattr` による存在確認を追加。

## 3. 実装上の工夫・変更点

- **仕様との整合性**: 指示書通りのGPIO 17, 27, 22を使用。
- **点滅の滑らかさ**: `gpiozero` の `blink` を使用することで、コードの可読性を保ちつつ、滑らかな点滅を実現。

## 4. 次の担当者への申し送り事項

- **ハードウェア変更時の注意**: LEDをコモンアノード（共通陽極）に変更する場合は、`LedController` の `RGBLED` インスタンス化時に `active_high=False` を指定する必要があります。現在はコモンカソード（共通陰極）に合わせて `default(True)` で動作しています。
- **ステート追加時**: 新しい内部状態（例: エラー状態、低電力状態など）を `voice_loop.py` に追加する場合は、`led_controller.py` の `set_state` メソッドに新しい色・パターンを追加してください。
