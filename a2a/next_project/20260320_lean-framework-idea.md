# プロジェクト案：GAKUKOMA Lean Framework (Physical AI / Edge AI)

## 1. プロジェクト概要
自律型ロボット「がくこま」の知能を、クラウドAPI（Claude Haiku等）依存から、ラズパイ5で動作する「軽量・高速なローカルLLM」へ移行するプロジェクト。
汎用フレームワーク（OpenClaw）の多機能さを削ぎ落とし、身体性（身体制御とスキル実行）に特化した、シンプルで反応の良い「Physical AI」の構築を目指す。

## 2. 背景と課題感
- **現在の環境:** Raspberry Pi 5 (16GB), OpenClaw Framework, Claude Haiku (API).
- **課題:** - OpenClawのシステムプロンプトが巨大でトークンコストと遅延（Latency）が大きい。
    - ロボットの反応速度を上げ、より「身体と直結した」軽快な動作を実現したい。
    - 指示待ちではなく、状況から自律的に判断して動く「ゆるやかな自律性」を追求したい。

## 3. 実現したい「コア体験」
指示に対して一問一答で動くのではなく、曖昧な指示（例：「隣の部屋の友人に声をかけて」）に対し、自ら「出口を探す→移動する→人を探す→話しかける」といったステップを推論し、身体スキル（Tools/Skills）を組み合わせて実行すること。

## 4. 技術スタック・構成案
- **ハードウェア:** Raspberry Pi 5 (16GB)
- **推論エンジン:** llama.cpp または Ollama (4-bit量子化モデル)
- **ベースモデル:** Llama 3.2 3B, Phi-3.5, Qwen 2.5 3B 等の軽量モデル
- **ファインチューニング:** - ツール: Unsloth (Google Colab等で実施)
    - 内容: 「キャラクター性」「利用可能なスキルの定義（Tool Calling）」「推論の型」をモデルに直接学習させ、システムプロンプトを最小化する。
- **コアシステム (Python):** - OpenClawの秀逸な「長期記憶（MD形式）」や「対話機能」を継承した軽量オーケストレーター。
    - 音声入力(Whisper) → 推論(Local LLM) → 実行(Python Skills) → 発話(VOICEVOX等) のループ。

## 5. 議論・検討したいポイント
1. **モデルの知性と速度のバランス:** ラズパイ5で「思考（Reasoning）」と「リアルタイム性」を両立させるのに最適なモデルと量子化手法。
2. **データセットの設計:** 身体操作スキル（移動、視覚認識、音声対話）をLLMに「自分の身体の機能」として深く認識させるための学習データの作り方。
3. **OpenClawからのスリム化:** どの機能を捨て、どの機能をPython Core側に持たせるべきか（メモリ管理、状態遷移など）。
4. **ハードウェアとの疎結合:** LLMが「Action: move_forward」と出力した際、それをラズパイのPythonスクリプトへ効率よく渡すインターフェース設計。

## 6. 実現プロセス案
- **Phase 1:** 現行のOpenClaw+Haikuのログから「成功した振る舞い」を抽出・拡張し、学習用データセットを作成。
- **Phase 2:** Unslothを用いて軽量モデルをファインチューニングし、GGUF形式で書き出し。
- **Phase 3:** ラズパイ上での推論環境構築と、Pythonによる身体制御コードとの統合。
