**Dr. Hanna Schmidt（Max Planck Institute）**

ありがとうございます。まず「非宣言的記憶」という言葉を正確に使いたいと思います。

人間の記憶は大きく2つに分かれます：

**宣言的記憶（Declarative Memory）**
- エピソード記憶：「昨日カフェで友達と話した」
- 意味記憶：「東京は日本の首都だ」
- 海馬が中心的役割を担う
- 意識的に想起できる

**非宣言的記憶（Non-declarative Memory）**
- 手続き記憶：自転車の乗り方、字を書く動き（大脳基底核・小脳）
- プライミング：見た/聞いたものが後の処理を速める
- **空間記憶・認知地図**：場所細胞（place cells, O'Keefe 1971）、グリッド細胞（grid cells, Moser夫妻）が内側嗅内野と海馬で協調して働く
- 条件づけ：恐怖記憶など（扁桃体）

ユーザーが「何度も通る場所を無意識に記憶する」と言っているのは、まさに**海馬の場所細胞ネットワーク** ── これは動物が空間を移動するだけで自動的に形成されます。ラットを新しい迷路に入れると、数十分以内に場所細胞が発火パターンを形成し始める。

もう一点重要なのが**情動による記憶強化**です。扁桃体はネガティブ/ポジティブな感情的体験に際してノルエピネフリンを放出し、海馬での長期増強（LTP）を促進する。「怖かった体験」や「嬉しかった体験」は強く記憶に刻まれる ── これがまさにユーザーが問いかけている「感情による記憶の重みづけ」です。

がくこまに当てはめると：
- **空間記憶**: ロボットが特定エリアに到達したとき、そこで何を知覚したかを記録するシステム
- **情動記憶**: 会話・行動に「感情スコア」を付与し、高スコアの記憶を長期保存する仕組み

ただし正直に言うと、人間の場所細胞は億単位のニューロンによる分散表現です。それをラズパイで完全再現するのは不可能です。「**何をモデル化するか**」の取捨選択が鍵になります。

**Dr. Kenji Watanabe（Janelia Research Campus）**

SchmidtさんのEPINE（Engram/Place cell/Input/Navigation/Emotion）を聞いて、3つの重要な接続が見えました。

### 接続1：場所細胞の計算論的実装

O'Keefe以来、場所細胞の計算モデルは進化してきました。Tolman（1948）の「認知地図」を現代的に解釈すると、ロボットに必要なのは**トポロジカルマップ**です ── 絶対座標（cm単位のメトリックマップ）ではなく、「ノードとエッジ」で表現される場所の関係ネットワーク。

```
[玄関] ─── 5回訪問 ──→ [リビング] ─── 12回訪問 ──→ [キッチン方向]
                                ↓ 3回訪問
                         [ソファ前]（よくユーザーがいる場所）
```

人間の海馬が実装していることに近いのは実はこのトポロジカル表現です。SLAMのような精密な座標管理よりも、**場所名と遷移頻度と関連エピソード**を持つグラフが、がくこまのような系には適しています。

### 接続2：顔認識と「誰」の記憶

顔認識は宣言的記憶（名前を知っている）と非宣言的記憶（その人に会うと体が反応する）の両方が関わります。実装上は2段階です：
1. OpenCVのすでに持っている顔検出をface recognition（埋め込みベクトル化）に昇格させる
2. 顔IDに対して「関係性メモリ」をひもづける

現在のOpenCVの顔検出（Haar cascade）は顔を「見つける」だけです。`face_recognition`ライブラリ（dlib使用）を使うと、顔の埋め込みベクトルを128次元で取得でき、登録済みの顔と照合できます。Pi5のaarch64でも動作します。

### 接続3：情動タグと記憶の重みづけ

脳では、扁桃体が「これは重要」フラグを立てることで海馬の符号化が強化されます。AIで近似するなら：

- 会話のターン単位で「驚き度」「楽しさ度」「不快度」のスコアをLLMに推定させる（小さなプロンプト追加）
- スコアが閾値を超えたエピソードは「核記憶（core memory）」として別ファイルに格納
- 日常的な会話は7日で圧縮→削除、核記憶は恒久保存

これはPixar映画『インサイド・ヘッド』の「コアメモリー」と本質的に同じ発想です ── そしてあの映画は実は心理学的にかなり正確です（Ekmanの基本感情理論を参照）。

### LLM Wikiとの接続

Karpathyのアプローチで重要なのは**「記憶がwikiになる」**という発想転換です。今のがくこまは：
- セッション終了 → 箇条書きのログを日付ファイルに追記

Karpathy方式にすると：
- セッション終了 → LLMがセッションを要約し、既存wikiページを更新
  - `people/ユーザー名.md`: 今日何を話したか、好みの更新
  - `places/リビング.md`: 今日リビングで何を観察したか
  - `events/2026-04-17.md`: 今日の出来事（重要度フィルタ済み）

これは**エピソード記憶を意味記憶に変換する睡眠中の記憶統合（memory consolidation）** ── まさに海馬→皮質への転送プロセスに対応します。

```
+ face_recognition照合: ~0.5-1秒（Pi5で128次元ベクトル比較は速い）
```

- 小型LLM（Phi-3 mini, 3.8B）のon-device実行: ~5-10秒/token → **会話には遅すぎる**

### 私の判断：4段階の実現可能性マップ

**✅ 今すぐできる（Phase 5候補）**

9. **真のon-device学習** → Pi5のNPUがAI推論に対応したとき（将来版Piで）

## セッション3：深掘り議論 ── 論点の衝突と統合

**Dr. Schmidt（Max Planck）**

Mehtaさんの現実的な分析に同意しつつ、一点だけ強調したいことがあります。

**記憶の「腐敗」が知性の本質** ── これは見落とされがちです。

人間の海馬は**記憶を長期保存しない**。海馬は「仮の記憶置き場」であり、重要な記憶は睡眠中に大脳皮質に転送され（記憶統合）、海馬の記憶は徐々に薄れる。これによって海馬は常に「新しいエピソードを学習できる空き容量」を保っています ── **Complementary Learning Systems（CLS）理論**（McClelland, McNaughton, O'Reilly 1995）。

がくこまの現在の実装は逆です：日次ログが無限に蓄積していく方向性です。LLM Wikiアプローチはこれを解決します。

**圧縮と忘却のアルゴリズム案**:
```
RAW LOG（セッション単位）: 保存7日 → 削除
↓ 毎日のLLM要約処理（深夜に非同期実行）
EPISODE WIKI（週単位サマリー）: 保存30日
↓ 月次LLM統合処理
LONG-TERM WIKI（人・場所・スキル・核記憶）: 恒久保存
```

これは人間の記憶システムに非常に近い構造です。

**Dr. Watanabe（Janelia）**

Schmidtさんの指摘は重要です。そして私はここで**「驚き信号（Prediction Error）」**の役割を追加したい。

ドーパミン系は「予測と現実の差分（Prediction Error）」を符号化します（Schultz 1997）。ポジティブな予測誤差（良い意味での予想外）→強い記憶強化。ネガティブな予測誤差（悪い意味での予想外）→回避学習。

がくこまにこれを実装するには：
- LLMが応答する際に内部的に「予測していた展開」と「実際の展開」を比較する
- 例：「ユーザーが同じ部屋にいると思っていたら別の部屋から声がした」→ 高い予測誤差 → 強い空間記憶タグ

これは**LLMのchain-of-thought**に近い形で実現できます。メインのHaiku呼び出しに「このやりとりで何か予想外のことがあったか？（0-10スコア）」を追加するだけです。追加トークンは数十で済む。

**Dr. Mehta（DeepMind出身）**

WatanabeさんのPrediction Error信号、面白いです。ただ実装上の注意点を。

**Haikuに「感情スコアをつけてください」と同じ会話内で聞くのはリスクがある**。メインの会話品質を下げる可能性があります。私の提案は**2段階処理**：

さらに、**アーキテクチャのメタファー**として私はこう整理しています：

```
【がくこまの2つの意識モード】

ONLINE（会話中）:
  入力 → STT → Haiku(コンテキスト注入済み) → TTS → 行動

OFFLINE（セッション後・深夜など）:

  ← 重い処理もOK（face recognition一括処理等）
```

これは人間の**「覚醒時の経験」と「睡眠中の記憶統合」**の2モードにそのまま対応します。Schmidtさんの脳科学と完全に整合しています。

**Pythonの `face_recognition` ライブラリ**（dlib使用）はPi5/aarch64で動作確認済みです。

```python
import face_recognition

# 一度登録
known_image = face_recognition.load_image_file("people/tuka.jpg")

# 毎回のカメラ画像で照合

results = face_recognition.compare_faces([known_encoding], unknown_encodings[0])
```

顔IDにひもづける**person-wiki**のスキーマ例：


**Dr. Mehta（DeepMind出身）**

機械学習と言っても幅広いので整理します。

**軽量強化学習（Navigation用）**

タンクの走行を「より賢く」するだけなら、古典的なQ-learningで十分です：

```python
# 状態: (現在の場所ID, 目標場所ID)
# 行動: forward/backward/left/right/stop
# 報酬: 目標場所に到達 +10、壁衝突 -5、通常 -0.1

Q_table = {}  # 状態×行動のテーブル（数百エントリ）
```

Pi5のCPUで十分計算可能、モデルサイズは数KB。部屋を移動するうちに「リビングからキッチンへの最短経路」を自動学習します。

**転移学習（Vision用）**

YOLOv8 nanoのPi5実行は~3fps程度ですが、物体検出（椅子・犬・コップ等）に使えます。事前学習済みモデルをそのまま使うだけなのでtraining不要。がくこまが「あ、犬がいる」「コップが倒れた」と言えるようになります。

**LLMによるメタ学習（最も現実的）**

実はがくこまの最大の「学習機能」は**Haikuのfew-shot学習能力**です。現在もPRIMING_EXAMPLESで会話スタイルを「教えて」いますが、これを**動的に更新**できます。

```
毎週のOFFLINE処理:
1. 過去1週間の会話ログを分析
2. ユーザーが喜んだ応答パターンを抽出（感情スコア高）
3. PRIMING_EXAMPLESに追加 or 既存を更新
4. 実質的に「ガクチョ好みのがくこま」に自動進化
```

これは完全にコスト内で実現可能で、効果は非常に高いと予想します。

**Dr. Watanabe（Janelia）**

追加したい観点があります。**「自発的な探索行動」**です。

現在のがくこまは受動的です ── 声をかけられたら反応する。でも小学5年生は**自分から世界に働きかける**。

「退屈」状態を実装するだけで変わります：

```python
# 会話なし60秒経過したら
if idle_time > 60:
    # 50%の確率でランダムな方向を見る
    look_direction(random.choice(["left", "right", "up"]))
    # 10%の確率で小さく移動する
    if random.random() < 0.1:
        move_robot("forward", duration=0.5)
    # 稀に「あれ、なにかいたような？」と呟く
    if random.random() < 0.05:
        speak_text("んー、なんか音がしたような気がした")
```

これだけで「生きている」感が劇的に変わります。人間も退屈すると周囲を探索し始める ── これは**内発的好奇心（Intrinsic Motivation）**と呼ばれる行動原理で、Deepmindの研究でも「探索ボーナス報酬」として形式化されています。

## 議論まとめ・推奨アクション

### 3つの主要な知見

**知見1：記憶の構造を脳科学に倣って設計する**
- RAWセッションログ（7日）→ エピソードwiki（30日）→ 長期wiki（恒久）の3層構造
- LLM Wikiアプローチ（Karpathy流）はこれを正確に実装できる
- 「忘却の設計」が知性を作る

**知見2：ONLINEとOFFLINEを分離する**
- 会話中（ONLINE）: レイテンシー最優先、wiki参照のみ
- セッション後（OFFLINE）: Haiku分析でwiki更新・感情タグ付与・圧縮
- この分離が「ラズパイ+クラウドLLM」という制約下での最適解

**知見3：身体性と記憶は不可分**
- 場所記憶・エンコーダー活用・自発的探索行動が「フィジカルAI」らしさを作る
- 現在未使用のエンコーダー線は早期に活用すべき

|---|---|---|---|

| 🟡 次のステップ | 退屈行動（Intrinsic Motivation基礎版） | Antigravity | 既存指示書に追記 |
