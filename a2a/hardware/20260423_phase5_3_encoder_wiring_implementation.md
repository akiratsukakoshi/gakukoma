## 目的

YP100タンクシャーシのモーターエンコーダー線（現在未使用）をGPIOに接続し、移動距離・方向の定量計測を可能にする。

| GPIO | 用途 |
|---|---|
| GPIO2/3 | I2C（PCA9685サーボドライバ） |
| GPIO12/13 | PWM（TB6612FNGモーター速度） |

**接続可能な空きGPIO候補**: GPIO5, GPIO6, GPIO11, GPIO23

## Step 1: エンコーダー線の仕様確認

YP100の左右モーターからは各2本のエンコーダー線が出ている（黄・白が左、緑・青が右）。

**まず確認してほしいこと:**

配線前にモーター基板・ケーブルを目視して以下を特定する：

| 確認項目 | 確認方法 |
|---|---|
| 電源線かどうか | 赤/黒（またはV+/GND表記）は電源・GND |
| 信号線の本数 | 1本=ホールセンサー単相、2本=A/B二相エンコーダー |
| 動作電圧 | 3.3V対応か5V専用かを基板・データシートで確認 |

**色の一般的な割り当て（YP100の傾向）:**
- ホールセンサー式（多い）: 電源・GND・信号の3本/motor
- 二相エンコーダー: A相・B相・電源・GNDの4本/motor（ただし2本/motorに見える場合は電源共通）

4本配線（左2本＋右2本）であれば：
- 黄=左A相信号、白=左B相信号（またはGND）
- 緑=右A相信号、青=右B相信号（またはGND）

**テスター使用確認手順:**
1. 電源OFF状態でテスターの抵抗測定モードに
2. 各線ペアの抵抗値を測定
3. 0Ω（ショート）→ GND共通線の可能性
4. 数kΩ→ プルアップ抵抗内蔵のホールセンサー信号線

---

## Step 2: 電源・GND接続

ホールセンサー方式であれば電源供給が必要：

| 電源電圧 | 接続先 |
|---|---|
| 3.3V対応 | Pi5 Pin 1 または Pin 17（3.3V出力） |
| 5V専用 | Pi5 Pin 2 または Pin 4（5V出力） |
| GND | Pi5 Pin 9 または Pin 14 など未使用GNDピン |

**注意**: 既にGNDピンが混んでいる場合はブレッドボードのGNDレールから引く。

---

## Step 3: 信号線のGPIO接続

確認した信号線を以下の推奨GPIOに接続：

| 信号 | 推奨GPIO | Piヘッダーピン |
|---|---|---|
| 左モーター A相 | GPIO5 | Pin 29 |
| 左モーター B相（あれば） | GPIO6 | Pin 31 |
| 右モーター A相 | GPIO11 | Pin 23 |
| 右モーター B相（あれば） | GPIO23 | Pin 16 |

**A相のみ（単相）の場合**: 左=GPIO5、右=GPIO11 の2本のみ接続。

**プルアップについて**:
- 3.3V系のオープンコレクタ出力の場合は10kΩのプルアップ抵抗（GPIO pin ↔ 3.3V間）を追加
- 内蔵プルアップ（gpiozero の `pull_up=True`）で代替可能な場合あり
- テスト動作で判断する

---

## Step 4: 動作確認テスト

接続後、以下のワンライナーでパルスが出ているか確認する：

```bash
python3 - <<'EOF'
from gpiozero import Button
from gpiozero.pins.lgpio import LGPIOFactory
import gpiozero, time

gpiozero.Device.pin_factory = LGPIOFactory()

count = [0]
def on_pulse():
    count[0] += 1
    print(f"パルス! 累計: {count[0]}")

enc = Button(5, pull_up=True)  # GPIO5（左モーターA相）
enc.when_pressed = on_pulse

print("手でタンクを前進させてください（10秒間）...")
time.sleep(10)
print(f"合計パルス数: {count[0]}")
EOF
```

**期待動作**: タンクを手で動かすとパルス数が増える

**もしパルスが出ない場合の確認事項**:
1. GNDが正しく接続されているか
2. 電源電圧が合っているか（3.3V vs 5V）
3. pull_up=False に変えると変化するか
