## 概要

Raspberry Pi 5 に USBマイクとスピーカーを接続し、
Linux上で音声の録音・再生が正常に動作する状態を構築する。

この作業はソフトウェア実装（Antigravity担当）の**前提条件**です。
完了報告書に「確認済みのマイクデバイスID」を明記してください。
Antigravityはそのデバイスを設定ファイルに使用します。

## 用意されているハードウェア（手配済み）

| アイテム | 仕様 |
|---------|------|
| Raspberry Pi 5 | 16GBモデル、アクティブクーラー取付済み |
| USBマイク | 単一指向性 USB接続 |
| スピーカー | 3.5mm または USB接続（仕様を本指示書末尾のチェック項目で確認）|

## 作業手順

### Step 1：現在の接続状態確認

Pi 5 を起動し、何も接続していない状態で以下を実行してください。

```bash
# 現在認識されているオーディオデバイス一覧
arecord -l
aplay -l
```

結果をメモしておきます（後で比較するため）。

---

### Step 2：USBマイクの接続

1. USBマイクを Raspberry Pi 5 の **USB-A ポート**に接続します
2. 接続後、再度デバイス一覧を確認します

```bash
arecord -l
```

**確認ポイント:**
- Step 1 の結果と比較し、**新しく追加されたデバイス**が USBマイクです
- `card X: ...USB...` または `card X: ...Microphone...` のような行が追加されているはず
- この `card X` の番号と `device Y` の番号をメモしてください

**例（番号は環境によって異なります）:**
```
card 1: Device [USB PnP Sound Device], device 0: USB Audio [USB Audio]
  → デバイスID は "hw:1,0"
```

---

### Step 3：マイク録音テスト

確認したデバイスIDで録音テストを行います。

```bash
# 5秒録音（"hw:X,Y" の X,Y は Step 2 で確認した番号に置き換えてください）
arecord -D hw:1,0 -f S16_LE -r 16000 -c 1 /tmp/mic_test.wav -d 5
```

コマンド実行後、**マイクに向かって「こんにちは、がくこまです」と話してください**。
5秒後に録音が終了します。

録音ファイルを再生して確認します：

```bash
aplay /tmp/mic_test.wav
```

**合格条件:** 自分の声がはっきり聞こえること。ノイズが極端に大きくないこと。

---

### Step 4：スピーカーの接続と確認

#### スピーカーの種類を確認する

**パターンA：USBスピーカー（電源不要タイプ）**

1. USBケーブルで Pi 5 に接続
2. `aplay -l` でデバイスが認識されているか確認
3. 以下のテスト音を再生

```bash
# テスト音（デバイス指定あり）
aplay -D hw:X,0 /usr/share/sounds/alsa/Front_Center.wav
# または
speaker-test -D hw:X,0 -t wav -c 1
```

**パターンB：3.5mmジャックスピーカー（アクティブスピーカー ＝ 電源内蔵）**

- 3.5mmプラグを Pi 5 の **ヘッドフォンジャック（3.5mmオーディオ端子）** に接続
- スピーカー自身の電源スイッチをONにする
- `aplay -l` で `card 0` の HDMI/Headphones を確認

```bash
# ヘッドフォンジャック経由でテスト
aplay -D plughw:0,0 /usr/share/sounds/alsa/Front_Center.wav
```

**パターンC：3.5mmジャックスピーカー（パッシブスピーカー ＝ 電源なし）**

- パッシブスピーカーの場合、Pi 5 のヘッドフォンジャックだけでは音量が非常に小さくなります
- この場合は **小型アンプモジュール**（PAM8403等、数百円）が必要です
- 接続後に音量を確認し、小さすぎる場合はアンプの追加を ClaudeCode に報告してください

**スピーカーの種類の見分け方:**
| 特徴 | 種類 |
|------|------|
| USB給電のみで動作、USB接続 | USBスピーカー（アンプ不要）|
| 電源アダプタ内蔵 or 充電式、3.5mm接続 | アクティブスピーカー（アンプ不要）|
| 電源なし、コードが2本（赤/黒）のみ | パッシブスピーカー（要アンプ）|

---

### Step 5：録音→再生の一気通貫テスト

マイクで録音して即座にスピーカーで再生します。

```bash
# マイクデバイスID（hw:1,0等）とスピーカーデバイスを確認した上で実行
arecord -D hw:1,0 -f S16_LE -r 16000 -c 1 /tmp/roundtrip.wav -d 5 && aplay /tmp/roundtrip.wav
```

「こんにちは」と話しかけ、5秒後にスピーカーから自分の声が返ってくれば成功です。

---

### Step 6：デフォルトデバイス設定（任意・推奨）

USBマイクをデフォルト録音デバイスに設定しておくと、`-D` オプションを省略できて便利です。

`/etc/asound.conf` を作成（または更新）：

```
# /etc/asound.conf
pcm.!default {
    type asym
    playback.pcm "plughw:0,0"    # スピーカーのcard番号に合わせる
    capture.pcm "plughw:1,0"     # USBマイクのcard番号に合わせる
}

ctl.!default {
    type hw
    card 0
}
```

**注意:** card番号は環境によって変わります。`arecord -l` / `aplay -l` の結果を見て正しい番号を設定してください。

---

### Step 7：音量調整

```bash
# アルサミキサーで音量調整
alsamixer
```

- F6でデバイスを選択
- カーソルキーで音量を調整（80-100%推奨）
- ESCで終了

設定を保存：

```bash
sudo alsactl store
```
