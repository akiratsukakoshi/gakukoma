## 背景・目的

パンチルト設定動的化（コード実装済み）のテスト中に、サーボ操作コマンドが無限ハングする事象が発生している。

調査の結果、以下が根本原因と特定された：

- **過去の180°パン操作**（断線が発生した際の操作）により、PCA9685の内部I2Cステートマシンが不正な状態（読み取り中断状態）で固まっている
- この状態ではPython側の `busio.I2C()` 初期化が無限ポーリングに入り、プロセスがハングする
- `i2cdetect -y 1` でアドレス 0x40 は見えるが、通信シーケンス自体は壊れている

本作業は以下2点を実施してこの問題を解消し、テスト実施可能な状態に戻すことが目的。

---

## 作業内容

### Task H-1: PCA9685 電源完全リセット

I2Cスレーブのステートマシンをリセットするため、電源を物理的に切断・再投入する。

**手順:**

1. `sudo shutdown -h now` でRaspberry Pi 5を安全にシャットダウンする
2. ACアダプタ（またはUSB-C電源）を抜いてPi5の電源を完全に切る
3. PCA9685のV+ピン（現在Pi5の**Pin 4（5V）**に接続）のジャンパーワイヤを抜く
4. PCA9685のVCC（3.3V制御電源、Pi5の**Pin 1（3.3V）**に接続）のジャンパーワイヤを抜く
5. 5〜10秒待つ（コンデンサ放電）
6. 配線を戻す前に **Task H-2** の配線修正を先に実施する（H-2完了後にH-1の6以降を続ける）
7. ジャンパーワイヤをすべて戻す
8. Pi5の電源を入れて起動する

### Task H-2: サーボ配線へのゆとり確保

パンが端（10°側・170°側）まで回転したときにジャンパーワイヤが引っ張られて断線・接触不良が起きないよう、物理的な余長（たるみ）を設ける。

**手順:**

1. 電源が切れた状態（H-1実施後）で作業する
2. パン軸（ch0）のサーボ3本線（赤・茶・橙）の配線を確認する
3. パン台座を手で左右に動かしながら、配線が引っ張られるポイントを確認する
4. ワイヤに**1〜2cm程度のたるみ（ループ）**を持たせ、可動範囲全域でテンションがかからないように配線を整える
5. 必要であればテープや結束バンドで配線をブラケット等に軽く固定し、動きに追従させる
6. チルト軸（ch1）も同様に確認し、上下動作時に引っ張られがないか確認する

### Task H-3: 復旧確認テスト

電源再投入後、以下をターミナルで実行して正常に復旧したことを確認する。

**3-1: I2Cバス確認**
```bash
i2cdetect -y 1
```
期待値: アドレス `0x40`（PCA9685）と `0x70`（All Call）が表示される

**3-2: サーボ動作テスト（コード経由）**
```bash
cd /home/tukapontas/gakukoma
bash tools/set_pan_tilt.sh 90 90
```
期待値: コマンドが数秒以内に `set_pan_tilt成功: pan=90° tilt=90°` を返す（ハングしない）

**3-3: 範囲テスト**
```bash
bash tools/set_pan_tilt.sh 180 90
```
期待値: `set_pan_tilt成功: pan=170° tilt=90°`（180°を入力しても170°にクランプされて正常動作）

**3-4: 左方向テスト**
```bash
bash tools/look_direction.sh left
```
期待値: `look_direction成功: pan=170° tilt=90°`
